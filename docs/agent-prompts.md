# Agent 提示词模板

下面这些提示词适合粘到 Codex、Claude Code、Qwen Code、Qoder CLI、CodeBuddy Code 等可以运行 shell 的 agent 里。

## 读取前的系统边界

请只分析我本机上我有权访问的微信聊天记录。先用命令导出到本地临时 JSON 文件，再基于 JSON 分析。不要上传原始聊天记录，不要把聊天记录写入仓库，不要输出手机号、地址、账号等可识别隐私信息。

## 快速了解一个群最近在聊什么

请运行：

```bash
opencli wx history "群名" --since 2026-01-01 -n 5000 --json > /tmp/wechat-group-history.json
```

然后读取 `/tmp/wechat-group-history.json`，总结这个群最近的主要话题、活跃人物、情绪趋势和高频梗。要求：

- 结论必须标注依据来自哪些发言模式，不要凭空判断。
- 如果证据不足，明确说“不足以判断”。
- 涉及具体个人时，尽量使用昵称，不输出现实身份信息。

## 分析某个人的群内人格和情感画像

请运行：

```bash
opencli wx history "群名" --since 2025-05-11 -n 20000 --json > /tmp/wechat-person-history.json
```

然后基于整个群的消息分析“某某”。注意：不要只看这个人自己说的话，也要看别人如何回应他、调侃他、向他求助、回避他或重复他的梗。输出用自然段，不要做成机械列表。

请覆盖：

- 群内角色：他在这个群里通常承担什么功能。
- 表达方式：他如何开玩笑、转移话题、维护面子、表达需求。
- 亲密关系推理：我们都是男生，群聊是男生熟人语境；请从他在同性朋友群里的表达方式、对异性/关系/承诺/暧昧/冲突的态度，二阶推理他在两性关系中可能呈现的互动模式。不要把推理写成事实。
- 证据方式：用概括证据，不要大段引用原聊天。

## 做成网页

请运行：

```bash
opencli wx history "群名" --since 2025-05-11 -n 30000 --json > /tmp/wechat-site-history.json
```

然后生成一个本地静态网站。每个人一个页面，包含群内角色、人格分析、情感画像、两性关系推理和证据切片。要求：

- 首页列出所有人和简短摘要。
- 每个页面使用自然段，不要把人格分析写成清单。
- 明确标注“这是基于群聊文本的推理，不是心理诊断”。
- 原始 JSON 不要放进公开目录。

## 关键词追踪

请运行：

```bash
opencli wx search "关键词" --since 2025-05-11 -n 500 --json > /tmp/wechat-keyword.json
```

分析这个关键词在群里被谁使用、什么时候变多、通常和哪些话题一起出现、它在群内关系里承担什么功能。
